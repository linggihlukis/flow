# Visual/UI Questions

Explore these areas. Not all will apply — use judgement.

## Layout & Structure
- How should this be laid out? (describe or reference an example)
- What's the information hierarchy? What should users notice first?
- Any specific grid, spacing, or density preferences?

## Components & Interactions
- What interactions should feel snappy vs deliberate?
- Any animations or transitions important here?
- How should loading states look?
- How should empty states look? (first-time user, no data)

## Responsiveness
- Which breakpoints matter? (mobile-first, desktop-first, both?)
- Any elements that should behave differently on mobile?

## Error & Edge States
- How should errors be surfaced to the user?
- What happens when something takes too long?
- What's the fallback if an action fails?

## Design Reference
- Any existing design file, screenshot, or reference to follow?
- Any component library or design system already in use?
